1 c++ API for the openPMD ray trace extension {#readme}
2 ======================================================
3 
4 This library provides a high-level API in C++ and Python to read and write openPMD files following the raytrace extension.
5 It uses the official openPMD-api at lower level.
6 
7 
8 Quantities
9 ------------------------------
10 The list of variables stored in the openPMD file as defined by the extension is reported in the following table:
11 
12 
13 | Variable | Comment | Units |
14 | --------- | ---------------- | ------------------------ |
15 | x,y,z | position | [] |
16 | dx,dy,dz | direction | (normalized velocity) |
17 | sx,sy,sz | polarization of non-photons | |
18 | sPolAx, sPolAy, sPolAz, sPolPh | s-polarization amplitude and phase of photons | |
19 | pPolAx, pPolAy, pPolAz, pPolPh | p-polarization amplitude of photons | |
20 | time | ray time w.r.t. ray generation | [ms] |
21 | wavelength | | [Ang] |
22 | weight | weight | |
23 | id | unique ID | |
24 | status | Alive or Dead | |

28 Dependencies
29 ------------------------------
30 This package depends on:
31  - [**cmake**](https://cmake.org) (3.21.1):
32  - [**openPMD-api**](https://www.openpmd.org/openPMD-api/) (0.12.1) used as base layer
33  - [**pybind11**](https://github.com/pybind/pybind11) (2.4.3) [optional, ON by default] for the python binding
34  - [**doxygen**](https://doxygen.nl) (1.8.14) [optional]
35 
36  If you do not have the openPMD-api installed on your system, this package downloads, compiles and install it for you. But installing it separately is highly advisable.
